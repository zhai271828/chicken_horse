# 联机功能方案

## 一、需求分析

### 目标
- 支持2-4人联机对战
- 一个玩家玩1P，另一个玩2P，甚至3P、4P
- 局域网或互联网联机

---

## 二、技术方案对比

### 方案1：局域网直连（WebSocket）⭐ 推荐

**原理**：
- 一台电脑作为"主机"运行游戏服务器
- 其他电脑通过浏览器连接到主机
- 使用 WebSocket 实现实时通信

**架构**：
```
主机电脑（运行服务器）
├── 游戏服务器（Node.js）
├── WebSocket 服务
└── 游戏逻辑（权威）

其他玩家电脑
├── 浏览器打开游戏
├── WebSocket 客户端
└── 渲染 + 输入发送
```

**优点**：
- 延迟低（局域网 < 5ms）
- 实现相对简单
- 不需要外部服务器
- 完全免费

**缺点**：
- 需要同一局域网
- 主机需要运行服务器程序
- 主机断线则游戏结束

**工作量**：40-60小时

---

### 方案2：Cloudflare Workers + Durable Objects

**原理**：
- 游戏服务器部署到 Cloudflare 边缘网络
- 全球玩家都可以连接
- 使用 Durable Objects 管理房间状态

**架构**：
```
Cloudflare 边缘网络
├── Workers（HTTP + WebSocket）
├── Durable Objects（房间管理）
└── D1 数据库（可选，存储排行榜等）

玩家A浏览器 ←→ WebSocket ←→ Cloudflare ←→ WebSocket ←→ 玩家B浏览器
```

**优点**：
- 全球可玩，不限局域网
- 无需自己管理服务器
- 自动扩展

**缺点**：
- 延迟较高（50-200ms）
- 免费额度有限
- 实现复杂度高

**工作量**：72-104小时
**成本**：约 $10/月（超出免费额度后）

---

### 方案3：WebRTC P2P

**原理**：
- 玩家之间直接连接（点对点）
- 只需要一个信令服务器交换连接信息
- 延迟最低

**架构**：
```
信令服务器（Cloudflare Workers）
├── 交换SDP信息
├── ICE候选协商
└── 房间匹配

玩家A ←→ WebRTC P2P ←→ 玩家B
```

**优点**：
- 延迟最低
- 服务器成本最低（只做信令）
- 适合2-4人小规模

**缺点**：
- NAT穿透可能失败
- 需要STUN/TURN服务器
- 实现最复杂

**工作量**：60-80小时

---

### 方案4：使用现成多人游戏服务

**选项**：
- **Photon Engine**：免费20CCU，适合小规模
- **Colyseus**：开源Node.js游戏服务器
- **Socket.io + Express**：简单易用

**优点**：
- 开箱即用
- 文档完善
- 社区支持

**缺点**：
- 可能有学习曲线
- 免费版有限制

**工作量**：30-50小时

---

## 三、推荐方案

### 最佳选择：方案1（局域网 WebSocket）

**理由**：
1. 你和同学在同一局域网，延迟最低
2. 实现最简单，工作量最小
3. 不需要外部服务器和费用
4. 可以快速原型验证

### 4.4 服务器核心逻辑

```javascript
// server/GameRoom.js
class GameRoom {
  constructor(id) {
    this.id = id;
    this.players = new Map();
    this.obstacles = [];
    this.phase = 'WAITING'; // WAITING, BUILD, RUN, RESULTS
    this.gameTime = 0;
  }

  addPlayer(ws, playerName) {
    const playerId = this.players.size + 1;
    this.players.set(playerId, {
      id: playerId,
      name: playerName,
      ws: ws,
      x: 100,
      y: 100,
      input: { left: false, right: false, jump: false }
    });
    return playerId;
  }

  update(deltaTime) {
    // 更新所有玩家状态
    for (const [id, player] of this.players) {
      // 应用输入
      this.applyInput(player);
      // 物理模拟
      this.updatePhysics(player, deltaTime);
    }
    // 广播状态给所有玩家
    this.broadcastState();
  }

  broadcastState() {
    const state = {
      type: 'room_state',
      players: [...this.players.values()].map(p => ({
        id: p.id,
        x: p.x,
        y: p.y,
        state: p.state
      })),
      obstacles: this.obstacles,
      gameTime: this.gameTime,
      phase: this.phase
    };

    for (const [id, player] of this.players) {
      player.ws.send(JSON.stringify(state));
    }
  }
}
```

### 4.5 客户端网络层

```javascript
// src/network/WebSocketClient.js
export class WebSocketClient {
  constructor() {
    this.ws = null;
    this.playerId = null;
    this.onStateUpdate = null;
  }

  connect(serverUrl) {
    this.ws = new WebSocket(serverUrl);

    this.ws.onopen = () => {
      console.log('Connected to server');
    };

    this.ws.onmessage = (event) => {
      const data = JSON.parse(event.data);
      this.handleMessage(data);
    };

    this.ws.onclose = () => {
      console.log('Disconnected');
    };
  }

  handleMessage(data) {
    switch (data.type) {
      case 'room_state':
        if (this.onStateUpdate) {
          this.onStateUpdate(data);
        }
        break;
      case 'player_joined':
        console.log(`${data.player.name} joined`);
        break;
    }
  }

  sendInput(input) {
    if (this.ws && this.ws.readyState === WebSocket.OPEN) {
      this.ws.send(JSON.stringify({
        type: 'player_input',
        input: input
      }));
    }
  }

  placeObstacle(obstacle) {
    if (this.ws && this.ws.readyState === WebSocket.OPEN) {
      this.ws.send(JSON.stringify({
        type: 'place_obstacle',
        obstacle: obstacle
      }));
    }
  }
}
```
