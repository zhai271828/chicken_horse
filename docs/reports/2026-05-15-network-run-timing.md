# 联机时间流速修复报告

日期：2026-05-15

## 问题

- 联机竞速阶段由房主客户端本地推进。
- 旧逻辑在 `NetworkRunState` 中使用 `setInterval + dt <= 100ms` 的单次更新。
- 当房主窗口失焦或浏览器把后台定时器降频到约 1 秒一次时，真实过去了 1000ms，但模拟只前进 100ms，导致第二轮以后更容易体感到“时间流速特别慢”。

## 处理

- 将房主联机回合循环改为固定步进追赶模式。
- 每次定时器回调先累计真实流逝时间，再按 `1000 / 60ms` 的固定步长多次推进 `RunState.update()`。
- 继续保留广播节流，避免因为追赶步进而向客户端洪泛状态包。
- 在联机暂停和退出时清空累计器，避免恢复时一次性补跑错误时间。

## 结果

- 房主窗口短时失焦时，回合时间与障碍物时间轴不再因为 `dt` 被截断而明显慢放。
- 该修复改善了“切屏后整局像慢动作”的核心问题。

## 剩余风险

- 当前架构仍然是“房主浏览器权威模拟”，不是服务端权威物理。
- 如果浏览器/系统把后台页面彻底挂起、电脑休眠、或页面长时间没有调度，仍然无法做到绝对零影响。
- 要做到“无论任何人切屏都完全不受影响”，最终方案还是要把权威竞速模拟迁到真正独立的服务端或专用后台线程体系。
