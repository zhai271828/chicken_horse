# 称号系统排查与修复报告

日期：2026-05-15

## 发现

- “一步之遥 / 终点前死亡”存在误判。
  - `RunState` 在玩家到达终点后会把玩家的 `lifeState` 改成 `DEAD`。
  - 旧的近终点死亡统计只看“这一帧从 ALIVE 变成 DEAD”，没有排除“成功通关”的情况。
  - 结果就是：成功到终点也会被记成一次“终点前死亡”。
- “十格检测”没有按设计实现。
  - 设计文档要求：终点周围十个方块的圆形区域。
  - 旧代码实际是终点附近 `3 格矩形` 判定。
- 陷阱类型统计不完整。
  - 多个死亡入口没有把具体陷阱类型传给 `ScoreManager.recordDeath()`。
  - 这会让“花式死法收藏家 / trap_tester”之类依赖 `trapTypesKilled` 的统计不准。
- 地图称号的 MVP 统计口径有偏差。
  - 部分称号汇总逻辑用了跨整局的 `totalPoints`，和“按当前地图累计到换图为止”这一设计不一致。

## 修复

- 近终点死亡只在“真实死亡且仍处于 PLAYING 状态”时才计入。
- 近终点范围改为可配置的 `10 格圆形检测`。
- 为炸弹、火焰、激光、尖刺平台、尖刺碰撞、投射物击杀、爆炸二次伤害补齐陷阱类型上报。
- 地图称号进度和汇总里的 MVP 改为使用当前地图积分 `points`，不再混用全局 `totalPoints`。
- 补充 `RunState / ScoreManager / RespawnManager` 相关测试，覆盖：
  - 十格圆形检测
  - 通关不再误记为终点前死亡
  - 陷阱类型统计
  - 地图称号使用当前地图积分

## 结果

- “终点前死亡”称号不再把成功通关误算进去。
- 十格检测已按设计开启。
- 称号所依赖的死亡类型数据和地图阶段 MVP 统计口径更一致。
