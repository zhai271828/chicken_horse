### 核心架构说明

**状态机模式**：游戏使用状态机管理不同场景，状态流转：
```
启动 → 主菜单 → 角色选择 → 地图选择 → 建造阶段 → 运行阶段 → 结果展示 → 商店 → 建造阶段...
```

**实体组件模式**：
- `Obstacle` 是所有障碍物的基类
- 子类通过 `isSolid`（是否阻挡）和 `isHazard`（是否致命）定义行为
- `applyEffect()` 和 `preEffect()` 用于特殊效果

**物理系统**：
- `PhysicsSystem.js` 处理所有碰撞检测
- 支持地图瓦片碰撞、玩家间碰撞、障碍物碰撞

### 1.5 陷阱归属与积分规则

| 规则 | 说明 |
|------|------|
| 陷阱署名 | 每个陷阱记录放置者（`_placedBy`），跨回合保留 |
| 击杀加分 | 陷阱杀死非主人的玩家 → 给主人 +2 分 |
| 击杀归属 | Flame、Laser、Bomb、SpikePlatform、Cannon弹丸、Arrow箭矢 均支持 |
| 跨回合保留 | 地图不变时陷阱保留到下一回合，玩家可叠加放置新陷阱 |
| 换图清空 | 达到100分换地图时，所有陷阱和归属清空 |
