# 彩色金币特殊效果功能计划

## 结论

可以实现。

彩色金币完全可以扩展成“拾取后触发特殊状态”的系统，包括但不限于：

- 体型变大
- 体型变小
- 移动速度变快
- 移动速度变慢
- 跳跃能力变化
- `Shift` 冲刺
- 短时间无敌
- 短时间吸金币

如果要支持联机，建议把它做成“主机权威 + 客机表现同步”的能力系统，而不是让客机自己决定能力结果。

## 推荐设计

### 1. 新增状态层

建议增加一个独立的“能力效果层”，不要把彩色金币逻辑直接塞进 `Coin.js`。

推荐位置：

- `src/config/powerups/`
- `src/systems/PowerupManager.js`
- `src/models/PlayerPowerState.js`

### 2. 每个玩家维护一个能力状态

每个玩家需要一份独立的临时能力状态，例如：

```js
{
  activeEffects: [],
  moveSpeedScale: 1,
  jumpScale: 1,
  sizeScale: 1,
  dashEnabled: false,
  dashCooldownMs: 0,
  dashRemainingMs: 0,
}
```

### 3. 彩色金币只负责触发

彩色金币不直接修改玩家速度或体型，只做两件事：

1. 决定触发哪种效果
2. 把效果交给 `PowerupManager`

这样后续换更多效果时，不用动金币实体本身。

## 单机实现步骤

### 阶段一：基础效果框架

1. 增加 `PowerupManager`
2. 给 `Player` 增加可叠加的能力状态
3. 在 `RunState.update()` 中每帧更新能力持续时间
4. 在 `Player.update()` 前把能力修正应用到移动、跳跃、体型

### 阶段二：先做 4 个低风险效果

建议先做：

- 变大
- 变小
- 加速
- 减速

这四个最容易验证，也最不容易和陷阱逻辑冲突。

### 阶段三：再做 `Shift` 冲刺

冲刺建议单独实现，不要混在普通速度加成里。

建议规则：

- 只有拿到指定彩色金币效果后才能冲刺
- 按 `Shift` 触发
- 持续 `150ms ~ 250ms`
- 冷却 `1000ms ~ 1500ms`
- 冲刺期间禁用二次触发

推荐落点：

- `Player`：保存冲刺状态
- `HandleInput` / `NetworkRunState.getLocalInput()`：增加 `dash`
- `Player.horizontalMovement()`：冲刺时覆盖水平速度

## 联机实现步骤

### 1. 主机决定效果

彩色金币被吃到后：

1. 主机判定拾取成功
2. 主机决定触发哪一个能力
3. 主机把效果同步给所有客户端

不能让客机自己随机效果，否则主客机会分叉。

### 2. 新增网络字段

建议在联机快照里同步：

- 当前效果列表
- 每个效果剩余时间
- 冲刺是否可用
- 冲刺冷却
- 玩家当前体型倍率

### 3. 新增输入同步

如果做 `Shift` 冲刺，联机输入要多一个字段：

```js
{
  left: false,
  right: false,
  jump: false,
  dash: false,
}
```

主机收到 `dash` 后统一判定是否可触发。

## 主要风险

### 1. 体型变化会影响碰撞

如果玩家变大/变小，`w/h` 改变后会影响：

- 地图碰撞
- 陷阱碰撞
- 终点判定
- 出生点是否卡墙

建议做法：

- 用“视觉缩放”和“碰撞箱缩放”分离
- 第一版先只改视觉和少量碰撞倍率
- 真正修改碰撞箱前，先验证不会把玩家卡进地形

### 2. 冲刺会影响陷阱判定

冲刺速度一旦过高，可能出现：

- 穿墙
- 穿刺
- 穿过终点不触发

建议做法：

- 保持冲刺速度适中
- 不直接瞬移
- 继续走现有逐帧碰撞流程

### 3. 联机下客机会更容易抖动

如果客机本地先显示冲刺，再被主机纠正，就会抖。

建议做法：

- 主机权威
- 客机只做轻量预测
- 关键状态以主机同步为准

## 推荐开发顺序

1. 单机版做 `PowerupManager`
2. 单机版先实现“变大 / 变小 / 加速 / 减速”
3. 单机版补 UI 提示和特效
4. 单机版加入 `Shift` 冲刺
5. 联机协议增加效果同步
6. 联机协议增加 `dash` 输入
7. 联机状态下做主客机一致性测试

## 涉及文件

建议主要改这些位置：

- `src/entities/Coin.js`
- `src/entities/Player.js`
- `src/states/RunState.js`
- `src/states/NetworkRunState.js`
- `src/systems/HandleInput.js`
- `src/network/NetworkManager.js`
- `workers/src/GameRoom.js`

## 建议

第一版不要一次做很多彩色金币效果。

最稳的顺序是：

1. 加速
2. 减速
3. 变大
4. 变小
5. `Shift` 冲刺

这样最容易定位问题，也最容易看清楚哪一类效果会破坏现有跑酷和陷阱节奏。
