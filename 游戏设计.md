
# 本地版游戏流程详解

## 一、整体流程概览

游戏采用状态机模式，完整流程如下：

```
启动(BOOT) → 主菜单(MENU) → 角色选择(CHAR_SELECT) → 地图选择(WALK_MAP) → 教程(TUTORIAL)
    → 陷阱布置(BUILD) → 比赛跑酷(RUN) → 结算(RESULTS) → 商店(SHOP) → 陷阱布置(BUILD) → ...
```

**核心循环**：BUILD → RUN → RESULTS → SHOP → BUILD → ...（每回合重复）

**地图切换条件**：当任意玩家累计积分达到100分时，切换到下一张地图（森林 ↔ 冰原），流程从 BUILD 重新开始。

## 二、各阶段详细说明

### 2.1 启动阶段（BOOT）

- 显示加载动画："加载中..."
- 显示课程标识："COMSM0166 -- Group 20"
- 1.5秒后自动跳转到主菜单

### 2.2 主菜单（MENU）

- 标题："The Incredible ChickenBunny"
- 动态提示文字："[ 按空格键开始游戏 ]"（带呼吸灯效果）
- 三个功能按钮：
  - **教程**（黄色）→ 进入教程页面
  - **联机**（蓝色）→ 进入联机大厅（本地版不使用）
  - **设置**（粉色）→ 打开设置面板
- 设置面板内容：
  - 显示模式切换（适应/拉伸）
  - AI地图生成开关
  - API密钥输入框
- **按空格键** → 进入角色选择

### 2.3 角色选择（CHAR_SELECT）

分两个阶段：

**阶段一：选择人数**
- 可选 2、3、4 人

**阶段二：选择角色和昵称**
- 按顺序选择：P1 先选，然后 P2，依此类推
- 四个可选角色（纯外观差异，能力完全相同）：

| 角色 | 中文名 | 标语 | 操作方式 |
|------|--------|------|----------|
| Chick（小鸡） | 小鸡 | Ready to race! | P1: 键盘WASD |
| Bunny（兔兔） | 兔兔 | Hop to victory! | P2: 键盘方向键 |
| Duck（企鹅） | 企鹅 | Waddle to win! | P3: 手柄1 |
| Polar Bear（北极熊） | 北极熊 | Chill champion! | P4: 手柄2 |

- 所有角色属性相同：速度3.2，跳跃力12，二段跳，重力0.7
- 角色不可重复选择（已选角色显示"已选"标签）
- 每个玩家有专属颜色标识：P1=蓝色，P2=橙色，P3=绿色，P4=粉色
- 每个玩家需输入昵称
- 全部选完后 → 进入地图选择

### 2.4 地图选择（WALK_MAP）

- 通过"走入门"的方式选择地图，非按钮选择
- 场景中有两扇传送门：
  - 左门：森林地图（map1）
  - 右门：冰原地图（map2）
- 玩家操控角色走向目标门，第一个进入的玩家决定地图
- 支持二段跳
- 进入门后 → 进入教程

### 2.5 教程（TUTORIAL）

两页弹窗式教程：

**第一页：操作说明**
- P1: WASD移动/跳跃
- P2: 方向键移动/跳跃
- 支持二段跳

**第二页：游戏流程**
- 角色选择 → 陷阱布置 → 比赛开始 → 道具商店

- SPACE/ENTER/点击 翻页
- ESC 返回主菜单
- 游戏中可从暂停菜单再次打开教程（返回暂停状态）

### 2.6 陷阱布置阶段（BUILD）

回合制布置陷阱，每位玩家轮流操作。

**第一回合**：跳过布置阶段（玩家没有陷阱道具），直接进入比赛。

**第二回合起**：每位玩家依次布置陷阱
- 陷阱面板：20种陷阱类型，3行×7列排列
- 操作方式：
  - 左键：放置陷阱
  - 右键：撤销上一次放置
  - R键：旋转炮台/风向
  - ENTER：结束当前玩家回合，进入下一位玩家或开始比赛
- 放置限制：
  - 不能放在地形内部（BFS洪水填充算法检测可放置区域）
  - 传送门需要配对放置（放置第一个后等待第二个）
- 陷阱所有权：每个陷阱记录放置者编号（`_placedBy`），用于击杀加分计算
- 开发者模式（D键）：无限放置

### 2.7 比赛跑酷阶段（RUN）

核心游戏阶段，所有玩家同时比赛。

**时间限制**：90秒

**游戏循环执行顺序**（每帧）：
1. 复活管理器更新
2. 时间管理器更新
3. 陷阱更新（移动平台移动、炮台发射、锯子旋转等）
4. 移动平台承载玩家
5. 物理前效果（冰面滑行、风力推动）
6. 玩家运动+物理碰撞（PhysicsSystem处理碰撞）
7. 终点检测
8. 物理后效果（坠落平台、弹跳垫、尖刺、传送门、火焰）
9. 金币收集
10. 投射物碰撞
11. 炸弹爆炸
12. 坠落死亡检测
13. 影子历史记录

**渲染顺序**：背景 → 瓦片地图 → 陷阱 → 金币 → 玩家 → HUD

**HUD显示内容**：
- 顶部：倒计时（<30秒变红）、回合数
- 每位玩家：金币数、钱包余额、背包按钮
- 背包按钮：打开道具库存面板
- 进度条：显示距离100分切换地图的进度
- 暂停菜单：继续、重新开始、教程、主菜单、开发者模式开关

**开发者模式快捷键**：
- K：自杀
- E：传送到终点
- T：冻结时间

**死亡与复活**：
- 玩家死亡后2000ms自动复活
- 陷阱击杀会给陷阱放置者加分

**计时结束**：检测是否有玩家卡关，然后进入结算

### 2.8 结算阶段（RESULTS）

每回合结束的计分板界面。

**显示内容**：
- 排名表，列：名次、玩家、状态、时间、死亡次数、金币、击杀数、积分、总积分、钱包
- 排名规则：通关者按时间升序排列，未通关者按金币数降序排列
- 第一名行高亮为金色



| 称号 | 条件 |显示或者补充说明

### 2.9 商店阶段（SHOP）

回合制购买道具，每位玩家轮流购买。

- 每回合显示8种随机道具（从20种中抽取，每回合打乱顺序）
- 特殊情况：卡关时（连续2回合无人通关），消除者（ERASER）作为第一件道具出现（免费，限1个）
- 每张道具卡片显示：图标、价格、已拥有数量、描述
- 悬停显示详细描述提示
- 购买按钮：扣除钱包余额，添加到库存
- 库存跨回合保留（存储在玩家实体中）
- Done/ENTER/S键：结束当前玩家回合

## 三、积分系统详解

### 3.1 积分获取规则

| 条件 | 积分 |
|------|------|
| 收集2个普通金币 | +1分 |
| 收集彩色金币（10金币价值） | +5分 |
| 陷阱击杀其他玩家 | +2分（给陷阱放置者） |
| 击杀所有其他玩家 | +3分 |
| 唯一过关者 | +5分 |
| 未死亡过关 | +3分 |
| 第一名到达终点 | +20分 |
| 第二名到达终点 | +10分 |
| 第三名到达终点 | +5分 |
| 第四名到达终点 | +2分 |

### 3.2 钱包系统

- 通关奖励和回合金币存入钱包
- 钱包余额用于在商店购买陷阱道具

### 3.3 地图切换

- 累计100分进入下一张地图
- 地图在森林（map1）和冰原（map2）之间交替
- 切换地图时清除所有陷阱所有权

### 3.4 卡关检测

- 连续2回合无人通关 → 触发卡关状态
- 卡关时商店出现消除者道具（免费，限1个）
- 消除者效果：移除3格范围内所有陷阱

## 四、金币系统

### 4.1 金币生成

- 金币放置在瓦片地图的有效位置
- 被陷阱阻挡的金币会重新分配到空闲瓦片
- 每回合随机1个金币被指定为**彩虹金币**（外观相同，收集时有彩色特效）

### 4.2 金币价值

| 类型 | 金币价值 | 积分价值 |
|------|----------|----------|
| 普通金币 | 1金币 | 0.5分（2金币=1分） |
| 彩虹金币 | 10金币 | 5分 |

## 五、地图系统

### 5.1 地图类型

| 地图 | 主题 | 特点 |
|------|------|------|
| map1 | 森林 | 绿色背景，8种随机背景图 |
| map2 | 冰原 | 蓝白色调，8种随机背景图 |

### 5.2 地图加载

- 使用 Tiled JSON 格式地图
- 通过 ChunkMapGenerator 从预定义的4×3网格中程序化组装
- 支持AI地图生成（需外部API密钥）
- 逻辑视口固定为960×544像素

### 5.3 瓦片类型

| 瓦片类型 | 说明 |
|----------|------|
| EMPTY | 空气 |
| SOLID | 实心方块（碰撞） |
| HALF | 单向平台（可从下方穿过） |
| SLOPE_UP | 上坡 |
| SLOPE_DOWN | 下坡 |
| SPIKE | 地图尖刺 |
| COIN | 金币生成点 |
| STARTPOINT | 出生点 |
| ENDPOINT | 终点（旗子） |

---

## 六、物理与碰撞系统

- 基于AABB（轴对齐包围盒）碰撞检测
- 支持瓦片碰撞：SOLID（四向碰撞）、HALF（单向平台，仅从上方碰撞）、SLOPE_UP/SLOPE_DOWN（斜坡）
- 玩家与玩家之间无碰撞（可重叠）
- 投射物（炮弹、弓箭）检测瓦片碰撞，碰到固体瓦片消失
- 传送器可传送投射物（方向随机360度，射速不变）
